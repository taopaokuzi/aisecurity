# AGENT 协作说明

## 目的

本文件用于约束并行 Agent 在本仓库中的协作方式，避免任务冲突、重复修改和状态失真。

## 必读顺序

任一 Agent 开始工作前，按以下顺序阅读：

1. `docs/tasks/TODO.md`
2. 自己要接手的 `docs/tasks/TASK-xxx-*.md`
3. 与该任务直接相关的实施文档或规格文档

## 状态唯一来源

- 任务状态的唯一来源是 `docs/tasks/TODO.md`
- 单任务文件不维护本地状态，只维护任务边界、范围和完成定义
- 若 `TODO.md` 与单任务文件描述冲突，以 `TODO.md` 的状态字段为准，以任务文件的范围定义为准

## 冻结文档

以下文档已冻结，Agent 不得直接修改：

- `docs/agent-identity-permission-prd.md`
- `docs/agent-identity-permission-srs.md`
- `docs/agent-identity-permission-technical-design.md`
- `docs/agent-identity-permission-development-guide.md`

## 任务领取规则

1. 只能从 `docs/tasks/TODO.md` 中领取任务。
2. 开始任务前，先把对应任务的 `实现状态` 改为 `IN_PROGRESS`。
3. 同时更新：
   - `执行人`
   - `最近更新`
   - `备注`
4. 只有当依赖任务满足 `实现状态=DONE` 且 `验收状态=PASS` 时，当前任务才允许启动。
5. 若任务依赖未完成，不得强行开始，应标记为 `BLOCKED` 或等待。

## 执行规则

1. 一个 Agent 一次只负责一个主任务。
2. 不要修改未领取任务的状态。
3. 不要覆盖其他 Agent 已经开始的文件改动。
4. 若发现文件已被其他线程修改，先兼容，不要回滚。
5. 所有实现必须以 `docs/tasks/TASK-xxx-*.md` 的范围和完成定义为准。
6. 修改前先查看任务文件中的 `建议写入范围` 和 `禁止改动范围`。
7. 如果必须越过任务写入范围，应先在 `TODO.md` 备注中说明原因。

## 完成规则

任务完成后，必须同步更新 `docs/tasks/TODO.md`：

- `实现状态` 改为 `DONE`
- `验收状态` 改为 `PENDING`
- `最近更新` 写当前日期
- `备注` 至少写明：
  - 修改了哪些文件
  - 跑了哪些命令
  - 测试是否通过
  - 是否还有遗留风险

任务通过验收后，再把 `验收状态` 改为 `PASS`。

## 阻塞规则

如果无法继续，必须把任务的 `实现状态` 改为 `BLOCKED`，并在备注中写明：

- 阻塞原因
- 被哪个任务阻塞
- 需要谁处理

如果任务实现完成但验收失败：

- `实现状态` 保持 `DONE`
- `验收状态` 改为 `FAIL`
- 在备注中写明失败原因和回修入口

## 输出要求

任一 Agent 完成任务时，至少应交付：

1. 代码或文档改动
2. 可复现的命令
3. 测试结果
4. 风险说明
